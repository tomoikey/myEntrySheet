## 学校の授業・研究以外での、あなたのこれまでのモノづくりに関して、具体的に記入してください。（内容の理解につながる画像や動画などの資料、WEB上で公開している制作物やポートフォリオがあれば閲覧可能なURLも記入してください。） 600文字
私は大学の友人らと「WEB勤怠管理アプリケーション」を開発しました。開発に踏み出した理由は、フリーランスプログラマーの友人の業務外事務作業による時間的損失が
あまりにも大きかったからです。時間的損失の具体的な原因として、打刻時間の記録のみならず、1ヶ月分の業務報告書の作成や請求書の発行まで全て自らで行っていた点が挙げられました。
その友人にバックエンド開発のメンターを務めてもらい、私はフロントエンド開発とバックエンド開発の双方を担当し開発に踏み出しました。
技術スタックとして、バックエンド開発はScalaとPlayFramework、フロントエンド開発はTypeScriptとReact、データベースはPostgreSQLを採用しました。
バックエンド開発では「処理速度を意識していて且つ実行時安全性の高い書き方か」という点、フロントエンド開発では
ReactでSPAとして実装しローディング時間の大幅な短縮化を図りつつ「ユーザーに配慮されているか」という点について意識して開発しました。
またコミット時にGitHub ActionsのCI/CDでアプリケーションの品質を維持しながら、バックエンドはGCPのk8sに、フロントエンドはVercelにデプロイされるようにymlファイルでワークフローを構築し、
開発生産性を高めるために全体工程で自動化できることは全て自動化するよう工夫しました。

## 上記に対して、技術的に困難な課題は何だったのか、そのために何をしたのかを記入してください 600文字
WEB勤怠管理システムの開発の中で困難を極めたのは3個あります。 
1つ目は、当初WEBアプリケーションの開発経験が全くなかったことです。そのためWEBアプリケーションがまずどのように動いているのか学習する必要がありました。YouTubeやGoogle検索を活用し
大まかな知識や概要をインプットした後、ドキュメントを参照しながら実際にTODOアプリケーションを開発して理解の定着に繋げました。その後Scalaを9年間書いている友人にメンターを務めてもらい、
現在の勤怠管理アプリケーション開発へと移行しました。
2つ目は、ユーザビリティの高いフロントエンド開発を進めるです。大きな問題として開発者にデザイン経験を持つ者がおらず、HTML/CSSの知識はあれど
構想を練ることや簡易的な雛形を作ることもできずにいました。そこで見やすく使いやすいUIに共通する点は何なのかを探るべく、デザイン性高いWEBサイトを複数ピックアップしました。
それらを元に見様見真似でベースを作り上げた結果、アイデアを実装に移しやすくなり最終的に以前とは比べものにならないUIの実装ができました。
3つ目は、インフラ技術です。HerokuのようなPaasを利用する選択肢もありましたが、インフラの学習の意味合いを兼ね備えてあえてIaasであるGCPを選択しました。
実際にコマンドラインからバックエンドリソースをk8sにデプロイし実際にアクセスできるようにすることや、証明書を発行させてhttps通信可能にすることなどは開発全体の中で最も困難を極めました。
始めは何が分からないのか分からない状態でしたが、GCPが提供している概念を一つずつ分解しドキュメントとGCPの説明動画を端から端まで読み漁り、約1ヶ月かけてアクセスできるようになりました。
